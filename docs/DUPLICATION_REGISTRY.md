# Duplication & Propagation Registry (WP2.5)

Behavior-preserving inventory of intentional and incidental duplication across the
codebase. Per Refactor Plan v3 §WP2.5, this is a **DECISION LEDGER, not a consolidation
plan** — the binding rule is: *"Do not consolidate protected logic merely because
arithmetic looks identical."* Each entry records: owner (canonical location), instances
(`file:line`), current semantics, guarding tests, and an explicit future decision.

**Scope of the WP that produced this file:** docs-only. Zero production/test/E2E code was
changed. No duplication was removed. Basis: `docs/SCAN_FINDINGS.md` +
`docs/scan/PHASE_*.md` (2026-07-03 line-by-line grounding scan) cross-checked against a
fresh code read (2026-07-10). Every instance below cites a live `file:line`.

**Decision vocabulary:**

- **KEEP SEPARATE** — semantics genuinely differ or logic is protected; do not merge.
- **DEFER (WPx)** — consolidation is plausible but belongs to a later, gated WP.
- **DO NOT CHANGE** — owner-gated behavior; touching it is a behavior change, not a refactor.
- **CONSOLIDATE (WPx)** — the *only* rows where merging is planned; each removes real drift,
  not arithmetic look-alikes.

Of the 14 items, **exactly one** (#14, schema manifests) is slated for consolidation, and
its owner is the already-planned WP2.6 schema registry. No row recommends merging protected
logic on arithmetic-similarity grounds.
