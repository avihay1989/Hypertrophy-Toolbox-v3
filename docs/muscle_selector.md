# Muscle Selector Documentation v3.1

Interactive SVG body diagram for choosing **priority muscles** in the Generate
Starter Plan modal (`/workout_plan`). Selecting muscles biases the generated
plan toward extra sets for them; selecting none = balanced volume across all.

A single MuscleMap anatomy figure is used (one for front, one for back) â€” there
is no Simple/Advanced toggle. The same generated SVGs also drive the Profile
coverage map.

## Features

- Inline SVG body map with anatomically-shaped, clickable muscle regions
- **Front / Back** tabs; left/right sides synchronize on hover and click
- Multi-select with hover / selected / partial visual states + tooltip
- Debug mode (shows muscle keys); "none selected" â‡’ target all muscles

## Body map

A single MuscleMap figure per side
(`id="body-{anterior,posterior}-hypertrophy-advanced"`), generated by
`scripts/build_musclemap_svgs.py`. Each region is **single-key** and ships
pre-canonicalized `data-canonical-muscles` (no runtime vendor-slug translation),
so clicking a region selects exactly its muscle.

## Selection model

`selectedMuscles` always stores **advanced leaf** keys. Region clicks flatten
through `SIMPLE_TO_ADVANCED_MAP` first (the machinery still supports multi-key
regions generically, though MuscleMap regions are single-key):

- `getCanonicalKeys(region)` â€” splits `data-canonical-muscles` into keys.
- `flattenToAdvancedChildren(simpleKeys)` â€” expands each to its leaf children
  (falling through to `[key]` for keys with no children).
- `regionVisualState(region)` â€” counts selected leaves â†’ `selected` / `partial`
  / `unselected`.
- `toggleRegion(region)` â€” mass add/remove of a region's leaves.

The legend is a flat list of muscle groups for the current side; clicking a
group toggles its leaves. `upper-back` and `hip-abductors` are legend-only (no
MuscleMap region).

## JavaScript API

```javascript
const selector = new MuscleSelector('muscle-selector-container', { debug: false });
selector.getSelectedMuscleIds();        // leaf keys, e.g. ['chest', 'biceps']
selector.getSelectedMuscleNames();      // ['Chest', 'Biceps']
selector.getSelectedMusclesForBackend();// backend names for priority_muscles
selector.setSelection(['chest']);
```

## Backend flow (`priority_muscles`)

1. **Frontend** (`app.js`) â€” `getSelectedMusclesForBackend()` â†’ first 2 muscles
   â†’ `priority_muscles` in the `/generate_starter_plan` body.
2. **Route** (`routes/workout_plan.py`) â€” forwards to `generate_starter_plan()`.
3. **Generator** (`utils/plan_generator.py`) â€” validates each name against
   `MUSCLE_TO_PATTERNS` (unrecognized are dropped), then
   `_apply_priority_muscle_boost()` adds volume for the matched muscles.

Empty selection â‡’ `priority_muscles: null` â‡’ balanced plan.

## State classes

| Class | Meaning |
|---|---|
| `.selected` | Region fully selected (green) |
| `.partial` | Some leaves of a group selected (yellow) |
| `.hover` | Pointer over the region (blue) |
| `.non-selectable` | Legacy no-op class; non-muscle parts live in `.body-outline` |
